This guide explains how to download, decrypt, and extract encrypted folders from SharePoint.

1Download from SharePoint

Download the encrypted file from SharePoint:

folder_name.tar.gz.age

Place it in a secure working directory.

2Obtain the Passphrase

You should receive the passphrase in a separate email from the sender.

Security Note: The passphrase email is separate from the file transfer for security reasons. Do not share or forward this passphrase.

3Decrypt + Extract

Method 1: One-Step Decryption (Recommended)

To decrypt and extract in a single command:

age -d folder_name.tar.gz.age | tar -xzf -

You will be prompted to enter the passphrase.

Command Explanation:

  • age -d - Decrypts the file and outputs to stdout
  • tar -xzf - - Extracts the compressed data from stdin

Method 2: Two-Step Decryption (Alternative)

If you prefer, decrypt first:

age -d -o folder_name.tar.gz folder_name.tar.gz.age

Then extract:

tar -xzf folder_name.tar.gz
After extraction, securely delete the .tar.gz file if required by your security policy.

4Verify and Clean Up

After successful extraction:

  • Verify the folder contents are correct and complete
  • Delete the encrypted .age file if required by policy
  • Delete the passphrase email securely
  • If using Method 2, delete the intermediate .tar.gz file

üîí Security Notes

  • Never store passphrases permanently in plaintext files
  • Delete temporary decrypted archives when finished
  • Keep the passphrase email separate from the encrypted file
  • Use the passphrase only for the intended file
  • Delete the passphrase email after successful decryption

‚ùì Troubleshooting

Wrong Passphrase Error

If you receive an error about incorrect passphrase:

  • Double-check you're using the correct passphrase from the email
  • Ensure there are no extra spaces when copying the passphrase
  • Contact the sender if the passphrase doesn't work

Extraction Errors

If extraction fails:

  • Ensure the encrypted file downloaded completely
  • Check available disk space
  • Verify file integrity if a checksum was provided
